# `<NuxtPage>`

 `<NuxtPage>` 是 Nuxt 提供的内置组件. `NuxtPage` 组件 需要被用于`/pages`顶级目录，或者嵌套页面中。

`NuxtPage` 是对 Vue Router 中的[`<RouterView>`]() 封装，`NuxtPage` 同样接收两个props `name` `route`.

- **name:** type: `string`

`RouterView` 根据 `name` 匹配路由列表中正确的名称来渲染相应的组件。


- **route:** type: `RouteLocationNormalized`

`route` 一个路由地址的所有组件都已被解析。

> **Nuxt 通过扫描和渲染`/pages`目录下的文件组件来解析`name` 和 `route`。**

除了`name`和`route` `NuxtPage` 还接收一个`pageKey` props。

- **pageKey:** type: `string` 或者 `function`

`pageKey` 在 `NuxtPge` 组件重新渲染时特别有用。

## 实例

例如，传一个`static` 值时，`Nuxt` 组件只能渲染一次。

```html
<!-- static key -->
<NuxtPage page-key=“static” />
```
或者， 在`/pages`目录中的组件里的`<script>` 部分 调用 `definepageMeta` 时设置 `key`

```js
<script setup>
definePageMeta({
  key: route => route.fullPath
})
</script>
```

## 自定义 Props

此外，在你需要向子组件传递props时`NuxtPage`还接收自定义的props，自定义的Props可以通过 Nuxt 应用的 `attrs` 来访问。

```html
<NuxtPage :foobar=“123” />
```

例如, 在上面的例子中,  `foobar` 的值可以通过`attrs.foobar` 获取.
